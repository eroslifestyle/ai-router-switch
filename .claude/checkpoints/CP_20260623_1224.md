## Key Decisions (44 Q&A, 14 round)

- **REGOLA OBBLIGATORIA non-interferenza**: il router tocca SOLO il backend. Mai `settings/`, mai `skill/`, mai `agent/`, mai `mcp/`, mai `tool/`, mai system prompt.
- **Gestione context NON nostra**: ogni modello gestisce il proprio context coi suoi settings. Eliminato routing `longContext/scenario`.
- **Gerarchia di priorità**: comando utente > `solo-X` rigido > `mixed` ottimizza > Pi orchestra (proxy si fa da parte).
- **Pi su porta dedicata**: solo compressione + fallback.
- **Compressione headroom ammessa solo byte-faithful**.
- **Modalità di default in produzione**: `minimax`.
